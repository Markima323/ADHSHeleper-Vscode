# Changelog

## 0.6.0

- 在代码编辑器右键菜单中加入“从此行开始学习”。
- 从光标所在行的行首截取到文件末尾，并沿用原有学习、朗读、AI 解释和本地缓存流程。

## 0.5.1

- 在 AI 选择框下方加入当前服务的 API 密钥管理框。
- 支持显示、隐藏、添加和修改 Gemini / DeepSeek 密钥，仍使用 VS Code SecretStorage 保存。

## 0.5.0

- 在“专注控制”面板中加入 Gemini / DeepSeek AI 服务下拉框。
- 新增 DeepSeek Chat Completions API，默认使用 `deepseek-v4-flash` 和非思考模式。
- Gemini 与 DeepSeek 的 API Key 分别保存在 VS Code SecretStorage。
- 卡片与行级解释按 AI 服务和模型分别缓存到 `D:\codeLearn`，切换后不会混用。

## 0.4.1

- 在“专注控制”面板中加入自动朗读开关，并与 VS Code 设置双向同步。

## 0.4.0

- 新增 `tts.autoPlay` 设置，可动态控制卡片自动朗读。
- 代码行悬停时显示蓝绿色解释圆点。
- 点击圆点请求 Gemini 行级解释，并在右侧可拖动悬浮窗中显示。
- 行级解释保存到 `D:\codeLearn` 并优先读取本地缓存。

## 0.3.0

- 学习代码按关键字、函数、方法、变量、类型、属性、字符串、数字、注释和运算符着色。
- 填空候选词继承其源码词元颜色，并跟随 VS Code 当前主题。
- 卡片格式升级时重建旧卡片，同时迁移已有 Gemini 解释。

## 0.2.5

- 上一张/下一张移动到卡片标题右侧。
- 英语重播和源码定位移动到页面底部。
- 删除填空区的任务标题、完成标题和数量标题。

## 0.2.4

- Gemini 解释在每个中文句号后换行，新旧本地解释都会应用。

## 0.2.3

- Gemini 提示词改为优先解释关键函数、方法、参数和返回值。
- Gemini 解释使用中文词语边界的部分加粗显示。
- 为本地解释加入提示词版本，旧解释只重新生成一次。

## 0.2.2

- 停止向 VS Code globalState 写入学习摘要，确保学习记录只位于 `D:\codeLearn`。
- 激活时清除旧版 `learningHistory` 摘要键。

## 0.2.1

- 将学习卡片、填空模型和 Gemini 解释保存到 `D:\codeLearn`。
- 相同文件、选区和源码再次学习时直接恢复记录并跳过 Gemini 请求。
- 本地恢复的解释显示来源标记，新增打开记录目录命令。

## 0.2.0

- 在卡片进度条和代码之间加入 Gemini 简洁解释。
- 使用 Gemini Interactions API、无状态请求和 minimal thinking。
- API Key 使用 VS Code SecretStorage，支持设置、删除和错误重试。
- 每个卡片会话内缓存解释，避免切换卡片时重复计费。

## 0.1.2

- 默认改为英语语音，每张学习卡片打开时自动朗读一次。
- 朗读期间仍可填写空位。
- 默认卡片缩小为最多 12 行，每张抽取 6 个空位。

## 0.1.1

- 在活动栏增加“ADHD Focus”控制面板，可直接切换部分加粗和开始学习。
- 快捷键或面板命令无法执行时显示明确原因。

## 0.1.0

- 可视区标识符与注释部分加粗。
- 选区、当前符号和文件分块学习。
- 德语系统朗读与最多 10 词点击填空。
- 本地会话摘要与可访问的键盘操作。
